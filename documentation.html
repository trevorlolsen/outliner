Local Outline

A private, encrypted outliner that lives entirely in your browser. This guide walks you through everything you can do with it.

1. Welcome

Local Outline is a hierarchical note-taking app inspired by tools like Dynalist. Everything you create stays on your own device, encrypted with your password. There is no account, no cloud, and no server — if you don't share your snapshot file, no one else can read your notes.

You can use it for:

  • Personal journals and daily notes
  • Project planning and checklists
  • Knowledge management with tags and cross-references
  • Code snippets with syntax highlighting
  • Pasted screenshots and reference images
  • Flashcards with spaced-repetition review
Important: Your password is the only key to your notes. If you forget it, nobody — not even the app's author — can help you recover the data. Use a password you can remember, and always keep at least one encrypted snapshot backed up somewhere safe.

2. Getting started

Creating your first vault

When you open the app for the first time you'll see the Vaults screen. Click + New vault and you'll be asked for:

  • Name — just a label so you can tell vaults apart (e.g. Personal, Work).
  • Password — minimum 8 characters. This encrypts everything in the vault.
  • Confirm password — type it again to catch typos.

Click Create. The vault opens and you're ready to start writing.

Unlocking a vault

Returning to the app shows the vault list. Click a vault and enter its password. After 15 minutes of inactivity (configurable), the vault locks itself automatically.

Multiple vaults

You can create as many vaults as you like on the same device. Each has its own password and its own contents — useful for separating, say, personal notes from work notes. The ⊞ button at the top of the sidebar returns you to the vault picker at any time.

3. Documents & the sidebar

Inside a vault, your work is organised into documents. Each document is a separate outline tree.

Creating documents

Click + New in the sidebar top row. A new untitled document opens; click the title to rename it.

Daily notes

Click the 📅 Today button to open or create today's daily note. The calendar below it lets you jump to any other date — daily notes are documents named by their date.

4. Working with outlines

Adding nodes

Click anywhere in the outline area or use the + Text button. Press Enter at the end of a node to add a new one below it.

Indenting and outdenting

Press Tab to indent (move under the previous sibling) and Shift+Tab to outdent. There is no nesting limit.

Moving nodes

Use Ctrl+↑ and Ctrl+↓ to move a node up or down. You can also drag nodes by their bullet.

Collapsing and expanding

Click the small triangle next to a bullet to collapse a subtree. Ctrl+. toggles the current node, and Ctrl+Shift+. expands every descendant.

Zooming in

Focus on a single subtree with Ctrl+]. The breadcrumb at the top shows where you are; click any crumb to zoom back out.

Deleting

Ctrl+Shift+⌫ deletes the current node and all of its children. There's also a context menu (right-click a bullet) for more selective operations.

Undo and redo

Ctrl+Z and Ctrl+Shift+Z. The app remembers the last 200 changes.

5. Node types

Every node has a kind. You can convert between kinds from the context menu.

Text

The default. Rich text with Markdown formatting (see §6).

Checkbox

A to-do item. Press Ctrl+Shift+C to insert one or convert a text node to a checkbox. Ctrl+/ toggles its checked state.

You can add a due date by typing @YYYY-MM-DD anywhere in the text, for example:

- [ ] Submit invoice @2026-05-22

Tasks due today, overdue, or due this week are surfaced in search and the sidebar counters.

Code block

Click + Code or convert a text node. Type the language right after the bullet (e.g. javascript, python, rust) — the editor switches to that language's syntax highlighting.

Supported languages: JavaScript, Python, Markdown, CSS, HTML, JSON, SQL, Rust, C / C++, Java, Julia, R. The editor below uses CodeMirror with the One Dark theme.

Image

Paste an image directly into the outline (Ctrl+V) and it becomes a node. Images are encrypted and stored inside the vault, never uploaded.

The maximum image size is 5 MB. Larger files are rejected with a notice.

7. Tags & links

Tags

Type #tagname anywhere to tag a node. Tags appear in the sidebar; click one to filter the document list to documents containing that tag.

Wiki links

Type [[ and start typing a document title — an autocomplete appears. Selecting one inserts [[Doc title]], which becomes a clickable link to that document.

Node references

You can also link to a specific node anywhere using ((node-id)). The easiest way to get the id is to copy a node link from the context menu.

Backlinks

When you zoom into a node or document, a panel at the bottom shows every place that links back to it.

8. Search

Press Ctrl+Shift+F (or click 🔍) to open the search overlay. It searches every node in every document in the current vault.

9. Spaced-repetition study

Any text node can be turned into a flashcard by starting it with the @card anchor. Cards live alongside the rest of your outline — you do not have to keep them in a separate document — and the app schedules them with a variant of the SM-2 spaced-repetition algorithm. This section covers every way to write a card, all the modifiers, and the full review flow.

9.1 The anchor

A card always begins with @card at the very start of the node (leading spaces are fine), optionally followed by a bracketed list of modifiers, then a single space, then the body of the card:

@card <body>
@card[reverse] <body>
@card[reverse,suspend] <body>

If the line does not start with @card, it is just a text node. If @card appears mid-line, it is just literal text. The anchor is what the app scans for when building the deck.

The fastest way to make a card is to type @cardSpace at the start of any node and then write the card. A pill badge appears next to the node as soon as the anchor is recognised.

9.2 The three card kinds

The kind is determined by what the body contains, not by an explicit choice. The same anchor produces a question card, an inline (front/back) card, or a cloze card automatically:

If the body…You get a…
Contains {{cN::…}}Cloze card
Has three or more top-level ::-separated partsMulti-side card (sides revealed sequentially)
Contains exactly one :: (and no cloze)Inline card (front/back)
Has neitherQuestion card (prompt only, no answer side)

9.3 Question cards

The simplest form. The whole body is the prompt; there is no separate answer to flip to.

@card Name the seven continents.
@card Recite the opening of Beowulf.
@card What is the Pythagorean theorem?

Use this for cards where there is no single canonical answer or where flipping doesn't make sense (recitations, lists you want to recall from memory, open-ended prompts). You still grade yourself in the session.

9.4 Inline cards (front :: back)

Put :: between the front and the back. Whitespace around the separator is trimmed.

@card Capital of France :: Paris
@card TLS stands for :: Transport Layer Security
@card What year was the moon landing? :: 1969
@card $e^{i\pi} + 1$ = :: $0$

Markdown and math work inside both halves — you can have a code snippet on the front and prose on the back, or images, or KaTeX equations.

Multi-line cards. Each node is a single line. For longer prompts or answers, break the card into nested children — the child nodes render as part of the card's prompt and answer respectively. (Or write a longer prose card as a question-card; flipping is optional.)

Multi-side cards (3 or more sides)

Add more :: separators to produce a card with as many sides as you like:

@card Subject :: Predicate :: Object
@card Latin :: English :: Example sentence :: Notes
@card Term :: Definition :: Mnemonic :: Source page :: Related terms

In the session you see the first side; each Space / Enter reveals the next one stacked below the previous. After the last side, the grade bar appears and a single rating applies to the whole card. Internally a multi-side card is one item with one schedule — the extra sides are just additional context that gets revealed before you grade.

The pill shows the side count (e.g. 🎴 4-side · due) so you can spot multi-side cards at a glance.

9.5 Cloze cards

Cloze deletion hides parts of a sentence. The syntax is {{cN::content}} — the c is literal, N is a number, and content is what gets hidden:

@card The capital of France is {{c1::Paris}}.

In the session you see the sentence with Paris blanked out, recall it, and grade yourself.

Multiple independent clozes

Give each cloze a different group number to schedule them independently. Each becomes its own card with its own review history.

@card {{c1::Mitochondria}} are the {{c2::powerhouse}} of the {{c3::cell}}.

That node produces three cards from one anchor: hide Mitochondria, hide powerhouse, hide cell. Each can be due on a different day.

Grouped clozes (reveal together)

Reuse the same group number to hide multiple regions together — they reveal as one and share a single review schedule.

@card In {{c1::1969}}, {{c1::Apollo 11}} landed on the Moon.

Here both blanks belong to group c1, so they form one card: the student must recall the year and the mission name in the same beat.

Hints

Add a second :: inside a cloze to supply a hint that stays visible:

@card The capital of France is {{c1::Paris::city}}.

In the session you see The capital of France is [city]. Hints are useful for disambiguating similar-looking prompts (e.g. several country/capital pairs in one paragraph).

Mixed example

A single cloze card combining independent groups, a grouped pair, and a hint:

@card {{c1::France}}'s capital is {{c1::Paris}}, while {{c2::Germany::country}}'s is {{c3::Berlin}}.

This produces three scheduled cards: a grouped c1 (France ↔ Paris together), a hinted c2 (Germany with the hint "country" shown), and a plain c3 (Berlin alone).

9.6 Modifiers

Modifiers go inside square brackets right after @card, comma-separated, with no spaces inside the brackets:

ModifierEffect
reverseFor an inline (2-side) card, schedules a second review item in the opposite direction. Each direction is tracked with its own SM-2 schedule, so the forward and reverse cards become due independently. The pill aggregates to whichever direction is soonest due, and clicking it shows both schedules side-by-side. Ignored on multi-side cards (with a console warning).
suspendKeeps the card in the deck but never schedules it for review. Useful when you want to put a card "on ice" without deleting it.

Examples:

@card[reverse] Bonjour :: Hello
@card[suspend] What is the cosine of 73 degrees?
@card[reverse,suspend] mitochondrion :: cell organelle
You can also suspend or unsuspend an existing card without editing its text — click the card's pill badge in the outline and choose Suspend / Unsuspend from the popover. That setting overrides the suspend modifier and is preserved when the card text is edited.

9.7 The card pill

Every recognised card shows a small pill badge next to the bullet. The pill tells you the card's state at a glance:

  • New — never reviewed yet
  • Due — ready for review now
  • An interval (3d, 2w) — how long until it's next due
  • Suspended — excluded from sessions
  • N-side — multi-side card with N sides (e.g. 🎴 4-side · due)

For reverse cards the pill aggregates the soonest due date across both directions; clicking it shows the forward and reverse schedules separately. For cloze cards it aggregates across every group the same way.

Click the pill to open a popover with three actions:

ActionWhat it does
Study just this cardOpens a one-card session
Suspend / UnsuspendToggles whether the card is scheduled
Reset statsWipes review history and interval; the card becomes "new" again. (Suspension state is kept.)

9.8 Starting a review session

There are four ways to launch a session:

  1. Sidebar due counter — if any cards are due in the vault, a counter appears in the sidebar; click it to study everything due.
  2. Card pill popover — "Study just this card" reviews exactly one card.
  3. Node context menu — right-click any node in the outline and choose 🎴 Study cards under here to review every card in that subtree.
  4. Command palette — press Ctrl+Shift+F, type >, then search for Study cards under selected node.

9.9 The session UI

Once a session opens:

  • The front of the first card appears. For multi-side cards, each Space / Enter reveals the next side stacked below.
  • Press Space or Enter (or click the card) to flip and reveal the answer (or, for multi-side, to reach the grade bar after the last side).
  • For reverse-companion cards, the queue contains both directions and the progress bar shows → forward or ← reverse so you know which one you're grading.
  • Rate yourself with one of the five buttons or their number keys:
ButtonKeyWhat it means
Again1You didn't recall it. The card returns in about 6 hours and the ease factor drops.
Hard2You got it but struggled. The interval grows slightly (×1.2) and ease drops a little.
Good3Comfortable recall. The interval grows by the current ease factor.
Easy4Trivial. The interval grows extra (× ease × 1.4) and ease nudges up.
Perfect5Instant recall with no hesitation. Largest interval growth (× ease × 1.8) and ease nudges up the most.

The session moves to the next due card automatically. When the queue is empty, it closes. You can close at any time with the × button or Esc; partial sessions still record the grades you gave.

9.11 Tips and recipes

Build a card from notes you already have

Convert the line in place: put your cursor at the start, type @card , then add a :: where the question/answer split is. The pill appears the moment the anchor parses.

One paragraph, many cards

Group related facts into a single cloze card with multiple groups. It keeps the context intact while still letting each fact be reviewed independently:

@card The {{c1::mitochondria}} of {{c2::eukaryotic}} cells produce {{c3::ATP}} via {{c4::oxidative phosphorylation}}.

Both directions of a translation

Use reverse on an inline card — one node, two cards:

@card[reverse] danke :: thank you

Bury a card you're not ready to review

Click its pill and choose Suspend. The card stays in the outline and remains visible; it simply won't appear in sessions until you unsuspend it.

Hide leading metadata

If a card needs context that shouldn't appear in the prompt, put the context in a parent node and the @card on a child:

- Chemistry / Bonding
  - @card Ionic bonds form between :: a metal and a nonmetal
  - @card Covalent bonds form between :: two nonmetals

Forgotten everything in a deck?

Use the pill's Reset stats action on each card to put it back to "new", or just keep grading them Again and the scheduler will gradually relearn your knowledge.

9.12 What's preserved when you back up or move

Card stats (interval, ease, review history, suspension state) are part of the node — encrypted snapshots preserve them exactly. Markdown export does not preserve them: the @card text round-trips, but a re-imported card starts as new. Use encrypted snapshots if you want your review schedule to survive a device transfer.

10. Memorization cards (@memo)

Where @card is for short prompt/answer recall, @memo is for memorising whole passages word-for-word — speeches, poems, prayers, scripture, code snippets you want to recall from memory. The app tracks mastery for every word individually and progressively blinds the ones you know best, so the prompt gets harder as you improve.

10.1 Anchor

Like cards, memo nodes start with an anchor:

@memo Four score and seven years ago our fathers brought forth
@memo[suspend] A passage to keep but not study right now

The body that follows is the passage. The only modifier today is suspend (reverse doesn't apply to memorisation). A 🧠 pill badge appears next to the node as soon as the anchor parses.

10.2 Word visibility modes

Each word in the passage carries one of four visibility modes that determine what you see during recall:

ModeWhat you seeWhen it's chosen
fullThe full wordWords you're still learning or that the algorithm has demoted
letterFirst letter, then underscoresDefault for new words; gives you a strong hint
blindUnderscores onlyWords you've mastered — the hardest level
lockedThe full word, alwaysManually-pinned words that should never be blinded (e.g. proper names you want to keep visible)

10.3 A review session

Click the 🧠 pill and choose Study just this memo. The review has three phases:

  1. Recall — the passage appears with each word rendered per its visibility mode. Try to recite it from memory. If you get stuck, click any word to peek at its full form; the click is remembered for the rest of this review.
  2. Reveal — press Space or Enter. The full passage appears below the masked version so you can compare. Grade yourself with one of the five buttons:
    • Again (1) — resets interval, demotes any word you peeked at or that has low mastery.
    • Hard / Good / Easy / Perfect (2–5) — grow the schedule and promote 2 / 4 / 8 of the highest-scoring words one step toward being blind (good → 2 words; easy → 4; perfect → 8).
  3. Blinding preview — the app shows you the suggested new state of every word, with words that are about to change highlighted with a dashed outline. Click Accept to commit, or Customise to fine-tune the suggestion by hand before saving.

The schedule (interval, ease, next due date) lives on the node alongside the per-word state.

10.4 Customising blinding

From the 🧠 pill popover, choose Edit blinding at any time (no review required) to open a customise view where each word cycles through full → letter → blind → locked → full on click. Use this to lock proper names that should always be visible, to hide tricky terms before you've earned the right with the scheduler, or to "warm up" a new memo by blinding common words you already know.

10.5 The 🧠 pill

States mirror the card pill:

  • 🧠 new — never reviewed yet
  • 🧠 due — ready for review now
  • 🧠 3d, 🧠 2w — days until next review
  • 🧠 💤 — suspended

The popover shows word counts (blind / letter / full), average mastery, and the full SM-2 stats. Actions: Study just this memo, Edit blinding, Suspend / Unsuspend, Reset stats.

10.6 Editing the passage

You can edit a memo's text any time. When you do, the per-word state is reconciled: words that still appear at the same position keep their visibility mode and mastery; new or moved words start fresh at letter with mastery 0. The reconciliation runs the next time you open the memo for review or customise.

10.7 Search filters

FilterMatches
@memoEvery memo node, regardless of state
@memo:dueMemos ready to review now

10.8 What's preserved when you back up or move

Encrypted snapshots round-trip the full memo state — per-word visibility, mastery, click counts, review history, and the schedule. Markdown export round-trips only the @memo anchor and the passage text; a re-imported memo starts fresh.

10.9 Tips

Start easy, ramp up

New memos start with every word in letter mode — a strong hint. As you grade Good / Easy / Perfect, the algorithm progressively blinds the words you find easiest to recall.

Lock proper names you don't want to memorise

If a passage contains a long proper name (a place, a person, a citation) that you want to keep as context but never hide, open Edit blinding, click each occurrence until it shows locked, then Save. Locked words are immune to all adaptive suggestions.

Burn-in a tough passage

If you're struggling with a section, click the words you can't recall during the recall phase. Clicked words are penalised in the next blinding pass — their mastery drops faster and they're more likely to be promoted back to full on the next Again.

Import from flash

Flash text-memory cards import directly as @memo nodes with their tokens and schedule preserved — no lossy conversion to question cards anymore.

11. Importing & exporting

Encrypted snapshot

The most important format. A snapshot is a single .json file containing every document, node, and image in the vault, encrypted with a password you choose at export time.

Export: Export → Encrypted snapshot, enter the vault password, save the file.

Import: from the Vaults screen click Import vault from snapshot…, pick the file, choose a name, enter the snapshot's password. A brand-new vault is created from its contents.

Markdown

Per-document import and export. Headings become document titles; - bullets become text nodes; - [x] / - [ ] become checkboxes; fenced code blocks become code nodes.

Markdown export does not include images — images appear as placeholder references. For a fully self-contained file, use HTML export instead.

HTML

Per-document. Exports a single self-contained HTML file with inline CSS, rendered Markdown, KaTeX math, and base64-embedded images. Great for sharing a read-only copy.

Print / PDF

Choose Export → Print / PDF to open the browser print dialog. Pick "Save as PDF" as the destination for a clean PDF copy of the active document; sidebars and modals are hidden automatically.

Importing from Flash

If you have an export from the sibling Flash memorization app, drop its .json file into the import flow. Standard and cloze cards are converted to @card nodes with their review history preserved.

12. Backups & transferring devices

Why backups matter

Your data lives in your browser's IndexedDB. Browsers can evict that storage if disk space runs low, and clearing site data (or uninstalling the browser) removes it permanently. Always keep at least one encrypted snapshot backed up outside the browser.

Auto-snapshot

Click Enable auto-snapshot in the sidebar to pick a folder. While the permission is granted, the app writes a fresh encrypted snapshot to that folder periodically. Re-enable if you switch folders or revoke the permission.

Pick a folder that is itself synced (e.g. a cloud-sync folder). That way your encrypted snapshots are automatically off-device, but they remain encrypted in transit.

Moving to a new device

  1. On the old device: Export → Encrypted snapshot, save the file.
  2. Transfer the .json file by any means (USB, cloud drive, email-to-self).
  3. On the new device: open the app, click Import vault from snapshot…, pick the file, enter its password.

The new device gets a fresh, independent vault containing the same data. Changes on either device remain local until you snapshot again.

13. Security & privacy

What's encrypted

Every document title, every node, every image. All encryption happens in your browser using AES-GCM with a key derived from your password (using Argon2id where available, PBKDF2 as a fallback).

What's not encrypted

Some app settings stored alongside the vault data are intentionally plaintext:

  • Your theme choice (light/dark)
  • The auto-lock timeout
  • The path to the auto-snapshot folder (handle only, not contents)

These contain no notes or personal information.

Zero knowledge

There is no server. Your password never leaves the device. Nobody can read your notes without your password, and there is no recovery mechanism.

Password change

Click the 🔑 in the sidebar to change your vault password. The internal encryption key isn't changed; only the way it is locked with your password. The change is instant — no re-encryption pass is needed.

Auto-lock

After 15 minutes of inactivity (configurable in settings) the vault locks itself and you'll need to re-enter the password. Closing the browser tab also locks the vault.

14. Settings

Themes

Click the 🌙 button in the sidebar to toggle light and dark themes. Your choice persists across sessions.

Auto-lock duration

Look for the setting in the sidebar / settings area. 0 disables auto-lock entirely (not recommended on shared devices).

Installing as an app (PWA)

On most browsers you can install Local Outline like a native app:

  • Chrome / Edge: click the install icon in the address bar.
  • Safari (iOS): Share → Add to Home Screen.
  • Firefox Android: menu → Install.

Installed, it launches in its own window with no browser chrome and works fully offline after the first load.

16. Troubleshooting

I forgot my password

There is no password recovery. The vault key is wrapped with your password and only you have it. If you've kept an encrypted snapshot somewhere and you remember the password you set when you exported it, you can import that snapshot. Otherwise the data is unrecoverable.

The app says "Incorrect password" but I'm sure it's right

Double-check the keyboard layout and Caps Lock. If you genuinely typed it correctly, the vault data on disk may be partially corrupted — try importing your most recent snapshot into a fresh vault.

I cleared site data and lost everything

Clearing browser site data wipes IndexedDB along with cookies and local storage. The vault data is gone unless you have an encrypted snapshot. This is one of the reasons auto-snapshot exists.

My snapshot file failed to import

Make sure you're using the same password you set when you exported the snapshot (not the current vault password — they're stored independently inside the snapshot). The error message "Incorrect password or corrupted snapshot" covers both cases without distinguishing them — this is intentional.

An image won't paste

Images over 5 MB are rejected. Resize or compress the image before pasting. If a non-image file is on the clipboard, the paste handler ignores it.

Auto-snapshot stopped working

Browsers can revoke the folder permission (e.g. after a restart). Click Enable auto-snapshot again to re-grant access.

The app won't open offline

Visit the app online once with a modern browser; the service worker caches everything needed. After that it works offline. If it still doesn't, check that your browser hasn't disabled service workers (some privacy settings do).

I want to migrate to or from another app

Markdown export works as a bridge to any other Markdown-based tool. For full fidelity (images, encryption, review state) only the snapshot format round-trips.

For the technical specification (encryption details, schema, internals) see outliner_technical_blueprint.md in the project directory.
